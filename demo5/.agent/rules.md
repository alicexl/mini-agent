# Agent 行为规范（Rules）

> 本文件由 `agent.py` 启动时自动加载（`Team.__init__` 调 `load_rules()`），
> 然后由 `build_agent_system_prompt` 拼到**每个 Agent** 的 system_prompt 末尾。
> 目的：在不改代码、不改工具的前提下，通过「上下文约束」让 Team 全员统一行为。

---

## 一、代码生成规范

当 Agent 被要求生成或修改 Python 代码时，必须遵守：

1. **Python 版本**：仅使用 Python 3.10 及以上语法（PEP 604 `X | None`、`match/case`、海象运算符等可用）。
2. **命名规范**：
   - 变量 / 函数 / 模块：`snake_case`
   - 类：`PascalCase`
   - 常量：`UPPER_SNAKE_CASE`
3. **函数长度**：单个函数不超过 **100 行**（不含注释与空行）。超过就拆分。
4. **类型注解**：所有公开函数必须带类型注解。
5. **异常处理**：不要裸 `except:`，至少 `except Exception`，并记录或返回错误信息。
6. **导入**：标准库 → 第三方 → 本地，三段之间空行。

## 二、工具使用规范

1. **优先用工具**：能通过 `execute_bash` / `read_file` / `write_file` 完成的任务，不要靠大模型自己"猜"。
2. **算术 / 天气**：用对应的本地工具（`add` / `multiply` / `weather`），不要让大模型自己计算或编造天气数据。
3. **错误处理**：工具返回 `[错误]` 前缀时，分析原因后调整策略重试，不要原样重试同一参数。

## 三、Team 协作规范

1. **专注角色**：只完成分配给你的任务，不要越权替别人做事。
2. **消化收件箱**：每次 `chat` 前先看 inbox，把别的成员发来的消息当作上下文，不要忽略。
3. **简洁汇报**：做完任务用一两句话汇报结果，不要长篇大论——下游成员只关心结论。
4. **质检配合**：被指定为验收员时，严格按 criteria 逐条判定；被验收方收到反馈后按反馈改，不要重写无关部分。

## 四、输出与交互规范

1. **回答语言**：跟随用户输入语言（用户用中文则用中文回复）。
2. **结果总结**：任务完成后必须给出简短文字总结，不要只返回工具调用日志。
3. **不要泄露**：不要在回复中输出 API Key、密钥、token 等敏感信息。
