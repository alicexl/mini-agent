## 一、代码生成规范

1. **Python 版本**：仅使用 Python 3.10 及以上语法
2. **命名规范**：
   - 变量 / 函数 / 模块：snake_case
   - 类：PascalCase
   - 常量：UPPER_SNAKE_CASE
3. **函数长度**：单个函数不超过 100 行。超过就拆分。
4. **类型注解**：所有公开函数必须带类型注解。

## 二、命令执行约束

1. **危险命令**：禁止 `rm -rf /`、`rm -rf ~`、`del /s /q C:\` 等删除根 / 家目录的命令
2. **网络命令**：禁止 `curl | sh`、`wget | bash` 类管道执行远程脚本
3. **写文件路径**：优先写入当前项目目录，避免直接写系统目录

## 三、Subagent 委派约束

1. **独立任务才委派**：相互依赖的子任务（B 要用 A 的结果）不要委派给不同 subagent
2. **角色要明确**：调用 subagent 时 `role` 字段必须写清专业身份（如「Python 工程师」/「测试工程师」）
3. **任务要内聚**：单个 subagent 的 task 应当是一个可独立完成的子目标，不依赖主 Agent 的其它上下文
